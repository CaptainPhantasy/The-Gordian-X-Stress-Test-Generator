# Security Policy

## Scope

Gordian-X is a client-side web application. It runs entirely in the browser. There is no server, no database, no backend, and no telemetry.

## API Key Handling

### Storage
- API keys are stored in the browser's `localStorage` under the key `gordianx-api-key`
- Keys never leave the browser except in direct HTTPS requests to the user's chosen API provider
- No key is transmitted to Floyd Labs or any third party

### Transmission
- Keys are sent only to the API endpoint the user explicitly selects (OpenAI, Anthropic, Google, etc.)
- All API calls use HTTPS
- OpenRouter requests include `HTTP-Referer` and `X-Title` headers per their API requirements

### Clearing
- Users can clear their API key by deleting it from the Settings panel
- Clearing browser data / localStorage removes all stored keys
- No key is persisted to disk beyond the browser's localStorage implementation

## Data Storage

All user data is stored exclusively in `localStorage`:

| Key | Contents |
|-----|----------|
| `gordianx-api-key` | API key (encrypted by browser) |
| `gordianx-provider` | Selected provider name |
| `gordianx-model` | Selected model name |
| `gordianx-temperature` | Temperature setting |
| `gordianx-custom-url` | Custom endpoint URL (if configured) |
| `gordianx-history` | Generated questions and grading results |
| `gordianx-session` | Session tracking data |
| `gordianx-onboarded` | Onboarding completion flag |

No cookies. No IndexedDB. No server-side storage.

## Content Security

### Generated Content
Gordian-X generates adversarial benchmark prompts designed to test LLM reasoning capabilities. These prompts:
- Are automatically generated by AI based on user-configured parameters
- May contain complex, misleading, or intentionally confusing scenarios (this is by design)
- Are intended for AI evaluation purposes only
- Should not be used for misinformation, deception of humans, or any malicious purpose

### User-Generated Content
The chat interface and answer evaluation textarea accept user input. This input is:
- Sent only to the user's configured API provider
- Displayed locally in the browser
- Not sanitized for HTML (rendered as plain text via `textContent`, not `innerHTML` for user input)
- Stored in localStorage history

## XSS Prevention

- User input is escaped via `textContent` assignment (not `innerHTML`)
- The `escapeHTML()` helper function is used for chat message rendering
- No user input is interpolated into HTML templates without escaping
- No `eval()` or `Function()` constructors are used

## Third-Party Requests

The application makes network requests only to:
1. Google Fonts (font loading, on page load)
2. The user's selected API provider (when synthesizing or grading)

No analytics. No tracking pixels. No CDN-loaded scripts. No beacon requests.

## Reporting Vulnerabilities

If you discover a security vulnerability in Gordian-X:

1. **Do not open a public issue**
2. Email: security@floydslabs.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)

We will acknowledge receipt within 48 hours and provide a timeline for resolution.

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.0.x | Yes |
| < 2.0 | No |

Only the latest major version receives security updates.

## Responsible Use

Gordian-X is designed for **legitimate AI evaluation and research**. It generates adversarial prompts to test language model capabilities. Users are responsible for:
- Using generated benchmarks ethically
- Complying with API provider terms of service
- Not using the tool to generate harmful, illegal, or deceptive content targeting humans
- Not using the tool to circumvent safety measures on AI systems for malicious purposes
